Rust相关的代码学习：
C++的C兼容负担正在导致C++本身的分裂：
（1）C的规范化和严格类型风格代码形成之后地位不可撼动，底层代码还是用这种方式处理；
（2）C++作为面向程序设计语言和多范式语言作为主体，一些性能的损失接受度越来越高，例如可选的GC等内容；
上述两个方面的原因，一种没有兼容负担的，强类型高效静态语言的需求越来越被需要，rust作为其中一个尝试，值得一试，并且通过对比也可以更好的探索C++的未来。

（3）最后使用cargo来进行项目的构建和运行测试等：
